<h1>Owl — Платформа для управления научными проектами и конференциями студентов</h1>
<b>Owl</b> — веб-приложение для организации подачи, модерации и публикации студенческих научных работ в рамках конференций. 
Система поддерживает многоуровневое взаимодействие между авторами, научными руководителями, модераторами и администраторами.<br>
<em>Статус: разработка на ранней стадии. Реализованы отдельные страницы фронтенда; бэкенд и интеграция находятся в процессе. Возможно, будет произведен пересмотр используемых технологий.</em>
<h2>Ключевые возможности (запланированы)</h2>
<ul>
  <li>Подача проектов (статей) с возможностью добавления соавторов и научного руководителя.</li>
  <li>Механизм согласия: все участники подтверждают обработку персональных данных.</li>
  <li>Гибкая система ролей:</li>
  <ul>
    <li><b>Пользователь</b> — подаёт и редактирует статьи;</li>
    <li><b>Модератор</b> — проверяет, принимает, отклоняет или отправляет на доработку;</li>
    <li><b>Администратор</b> — управляет всеми данными, текстами страниц, пользователями.</li>
  </ul>
  <li>Внутренний чат по каждой статье для обсуждения с модератором.</li>
  <li>Новостная лента на главной странице (редактируется админом/модератором).</li>
  <li>Формирование программы конференции и сборника материалов.</li>
  <li>Уведомления (в системе и по email через SMTP).</li>
</ul>
<h2>Архитектура</h2>
На данный момент архитектура системы находится в разработке.
<h2>основные сущности БД</h2>
<ul>
  <li><b>Человек</b> — персональные данные (ФИО, email, вуз и т.д.)</li>
  <li><b>Пользователь</b> — привязка человека к роли (один человек → несколько ролей)</li>
  <li><b>Конференция</b> — активное/архивное событие с датами и секциями</li>
  <li><b>Проект</b> (статья) — основная единица: файл, статус, участники, секция</li>
  <li><b>Новость</b> — заголовок, текст, автор, дата</li>
  <li><b>Сообщение</b> — в чате к проекту</li>
</ul>
